## 项目背景

**研究课题**：基于 CPET 数据，为老年心血管疾病患者（高血压/冠心病/心衰）构建运动"安全线"预测模型。

**核心方法（Stage 1B）**：
- **数据**：N=3,232（CTRL 1858 + HTN_no_EHT 738 + HTN_with_EHT 277 + EHT_only 359），87 个 CPET 指标（中文列名）
- **参考正常值**：条件分位数回归（QuantileRegressor + SplineTransformer，按年龄/性别/BMI 分层）
- **表型负担引擎**：Reserve 域 + Ventilatory 域，0/0.5/1.0 梯度评分
- **不稳定覆盖规则**：运动性高血压（EHT）严重/轻度分级 → 红色/黄色覆盖
- **结局锚点验证**：ElasticNet（交叉验证 AUC=0.564±0.016，测试 AUC=0.608）
- **置信度引擎**：4 维权重（completeness 0.25 / effort 0.20 / anchor 0.25 / validation 0.30），高置信 ≥0.80
- **最终安全区分布**：Green 19.7% / Yellow 25.6% / Red 29.7% / yellow_gray 24.9%；高置信 33.0%
- **构念效度**：Green 14.0% < Yellow 15.1% < Red 18.0%（EHT 率单调梯度，Cochran-Armitage p=0.025）

**Git 仓库**：`https://github.com/mrxachen/cpet-safety-line-stage1`（公开，已推送至 v2.7.0 前；本次提交含 v3.1.0 全部更新）

### 维度 2：方法学严谨性

请评估以下方法选择是否恰当，并指出任何潜在问题：

1. **参考正常值构建**：条件分位数回归（QuantileRegressor）是否是该场景的最优选择？与传统年龄分层参考值方法相比优劣如何？

2. **表型分区逻辑**：Reserve 域 + Ventilatory 域的 0/0.5/1.0 梯度评分是否有足够的文献支撑？EHT 覆盖规则的阈值设定是否合理？

3. **置信度引擎**：4 维权重（completeness/effort/anchor/validation）是否有方法学依据？AND 封顶逻辑是否解释清楚了？

4. **结局锚点验证**：AUC=0.608 作为验证证据是否充分？论文中是否过度推断了模型的预测能力？

5. **构念效度验证**：Green<Yellow<Red 的 EHT 率梯度（p=0.025）是否足以支撑构念效度声明？样本量是否足够？
